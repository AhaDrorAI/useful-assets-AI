Terminal Basics

What it is

A terminal is a text window for controlling the computer

You paste a command, press Enter, and read the result.

node --version

If Node is installed, you see a version like v24.15.0.

Windows

Open PowerShell

  1. Press Start.
  2. Type PowerShell.
  3. Open Windows PowerShell or Terminal.
  4. Do not use Run as Administrator unless official docs require it.
Mac

Open Terminal

  1. Press Command + Space.
  2. Type Terminal.
  3. Press Enter.
Safety ruleIf a command includes delete, remove, rm, reset, force, sudo, or asks for a secret key, stop and ask what it does before running it.

Install Checklist

Step 1

Install a code editor

  • VS Code is the simplest default.
  • Cursor is useful if you want an AI-native editor.
Step 2

Install Git

Download Git, then check:

git --version

Success: a line starts with git version.

Step 3

Install GitHub CLI and sign in

Download GitHub CLI, then run:

gh --version
gh auth login
gh auth status

Usually choose GitHub.com, HTTPS, and Login with a web browser.

Step 4

Install Node.js

Download Node.js LTS, then open a new terminal:

node --version
npm --version
Step 5

Install Claude Code CLI

Official docs: Claude Code setup

Windows PowerShell:

irm https://claude.ai/install.ps1 | iex

Mac:

curl -fsSL https://claude.ai/install.sh | bash

Check and start:

claude --version
claude doctor
claude
Claude Login

Sign in and choose an interface

The first claude run opens a browser sign-in. Inside Claude Code, use /login if you need to sign in again.

  • Terminal CLI: run claude inside the project folder.
  • Desktop app: download from claude.com/download, sign in, then open the Code tab. Desktop includes Code; install the CLI separately only if you want the claude terminal command.
  • Web: use claude.ai/code for Claude Code on the web or Remote Control workflows.
claude
claude --continue
claude --permission-mode plan
claude --permission-mode acceptEdits
Dangerous modeclaude --dangerously-skip-permissions is equivalent to --permission-mode bypassPermissions. Use only in isolated containers, VMs, or dev containers, not on a beginner's main machine.
Step 6

Install Codex CLI

Official docs: Codex CLI

npm i -g @openai/codex
codex

The first run prompts you to sign in with ChatGPT or an API key.

Codex Login

Sign in and choose an interface

codex login
codex login status
codex logout

Use device auth when a browser cannot open:

codex login --device-auth

Use API key login through stdin, not by pasting secrets into chat:

printenv OPENAI_API_KEY | codex login --with-api-key

Codex also has desktop and cloud paths:

  • Terminal CLI: codex
  • Desktop app: codex app opens or starts installing Codex Desktop on macOS/Windows.
  • Web: use chatgpt.com/codex when your plan includes Codex.
codex
codex --model gpt-5.4
codex --sandbox workspace-write --ask-for-approval on-request
codex --search
codex resume --last
Dangerous modecodex --dangerously-bypass-approvals-and-sandbox and codex --yolo bypass approvals and sandboxing. Use only inside an externally hardened runner.
Step 7

Install Bun

Bun installation docs

Windows PowerShell:

powershell -c "irm bun.sh/install.ps1|iex"

Mac:

curl -fsSL https://bun.com/install | bash
bun --version
Step 8

Optional project tools

  • Supabase CLI for database/auth/storage projects.
  • Netlify CLI for publishing sites.
  • Playwright for browser tests inside a project.

Final Check

Verify
git --version
gh auth status
node --version
npm --version
bun --version
claude --version
claude doctor
codex login status

If one command fails, stop and fix it before installing more tools.
