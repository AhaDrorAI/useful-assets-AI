אחרי התקנת Claude Code / Codex

בחרו את ריפו ה-AI coding שמתאים לרגע שבו אתם נמצאים

מדריך מעשי ליזמים ולאנשים שאינם מפתחים: מה להתקין, מה רק לקרוא, ומה לדחות עד שיהיה צורך אמיתי.

הכללהתקנה גלובלית רק למה שתשתמשו בו כמעט בכל פרויקט. התקנה לפרויקט רק למה שעוזר לפרויקט הזה. ריפו כהשראה אם צריך רק רעיונות.

מה זה ריפו ואיך "מתקינים" אותו?

ריפו (repository) הוא תיקיית קובצי קוד שמתארחת ב-GitHub. "להתקין ריפו" = להוריד אותו למחשב שלכם ולהגיד ל-Claude Code היכן הוא.

שיטה 1 — אוטומטית (מומלץ):
השתמשו ב-Prompt המוכן שבתחתית הדף — שלחו אותו ל-Claude Code והוא יתקין הכל בשבילכם.

שיטה 2 — ידנית (אם רוצים להבין מה קורה):

  1. לחצו על שם הריפו (הקישור הכחול)
  2. ב-GitHub, לחצו על הכפתור הירוק Code → העתיקו את ה-URL
  3. בטרמינל, כתבו: git clone [URL שהעתקתם]
  4. ספרו ל-Claude Code היכן התיקייה: "יש לי את הריפו [שם] בנתיב [נתיב]"

גלובלי vs. לפרויקט: "גלובלי" = מותקן פעם אחת ועובד בכל הפרויקטים. "לפרויקט" = מוסיפים לפרויקט ספציפי בלבד.

יועץ קצר

קטלוג ריפואים

להתחיל קטןקודם workflow guardrails. רק אחר כך memory, browser automation או platform מלא.
קודם פרויקט, אחר כך גלובלינסו skill או agent בתוך פרויקט אחד לפני שמקדמים אותו להתקנה גלובלית.
תמיד לבדוקכל workflow צריך להסתיים בבדיקה, screenshot, PR summary או רשימת סיכונים.

מילון מונחים

  • TDD — פיתוח מונחה-בדיקות (Test-Driven Development): כותבים בדיקות לפני הקוד
  • hooks — קוד שרץ אוטומטית לפני/אחרי פעולות מסוימות (למשל: לפני שמירה, לפני commit)
  • evals — הערכות — בדיקות אוטומטיות לאיכות תשובות AI
  • sandbox — סביבה מבודדת — Claude Code עובד שם בלי לפגוע בקבצים אמיתיים

Prompt מוכן להעתקה

I already installed Claude Code / Codex.
Help me choose the smallest useful agent setup for this project.

First inspect the project context and tell me:
1. Which workflow stage I am in
2. Which repos or skills are useful now
3. Which ones should be skipped for now
4. What should be installed globally vs project-level
5. How we will verify the setup

Do not install anything before explaining the reason and asking for approval.
