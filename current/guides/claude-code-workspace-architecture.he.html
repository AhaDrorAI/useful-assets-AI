הבעיה: Context Overflow

בלי מבנה — כך נראית השיחה

אתם מתחילים פרויקט. בהודעה 5 Claude מבין מה הוא צריך לעשות. בהודעה 15 הוא עדיין עובד. בהודעה 25 הוא כבר לא זוכר שיש קובץ CSS נפרד, ששינוי X לא אמור לפגוע ב-Y, ושהסגנון צריך להישאר עקבי. הוא מתחיל להמציא מחדש.

עם מבנה — Claude מחובר לאורך זמן

בתחילת כל שיחה Claude Code טוען את קובץ CLAUDE.md ואת מבנה התיקיות. הוא יודע: מי אתה, מה הפרויקט, אילו כלים בשימוש, מה הסגנון, ומה כבר הוחלט. אין צורך להסביר מחדש.

CLAUDE.md — זהות הפרויקט

קובץ טקסט אחד בשורש הפרויקט. Claude Code טוען אותו אוטומטית בתחילת כל שיחה. כאן תכתבו: מי אתם, מה הפרויקט, אילו כלים בשימוש, מה הסגנון, ומה Claude לא אמור לעשות לעולם.

זה כאילו נתתם ל-Claude "briefing" מלא — בלי שתצטרכו לחזור עליו כל פעם מחדש.

תבנית מומלצת

מה לכלול ב-CLAUDE.md

כיצד יוצרים קובץ CLAUDE.md:
ב-VS Code: לחצו ימני על תיקיית הפרויקט → New File → הקלידו CLAUDE.md → Enter.
אחר כך הדביקו את התוכן שלמטה ושמרו (Ctrl+S).

זהו קובץ Markdown רגיל. Claude Code קורא אותו בכל התחלת שיחה. דוגמה:

# הקשר הפרויקט

## מי אני
בעל עסק קטן, לא מפתח. מנהל קורסי אונליין ב-Teachable.

## מה הפרויקט
אתר Landing Page לקורס AI לסטארטאפים. מבוסס Next.js.

## כלים בשימוש
- GitHub: repo my-org/landing-ai
- Supabase: לאחסון רשומות leads
- Netlify: פרסום אוטומטי מ-main

## כללים חשובים
- אל תשנה את הסגנון הויזואלי בלי לשאול
- כל קובץ חדש → ספר לי מה עשית ולמה
- שמות קבצים תמיד באנגלית, kebab-case
Slash Command לדוגמה

יצירת slash command מותאם

כיצד יוצרים את התיקייה ב-VS Code:

  1. פתחו את תיקיית הפרויקט ב-VS Code
  2. בסרגל הצד, לחצו ימני על שם הפרויקט → New Folder → הקלידו .claude → Enter
  3. לחצו ימני על .claude → New Folder → הקלידו commands → Enter
  4. לחצו ימני על commands → New File → הקלידו review.md → Enter

שימו לב: תיקיות שמתחילות בנקודה הן נסתרות. VS Code מציג אותן תמיד, אבל ב-File Explorer רגיל הן לא נראות.

יוצרים קובץ בתיקייה .claude/commands/. לדוגמה, הקובץ review.md:

סקור את הקוד שכתבת בשיחה הזו.
בדוק:
1. האם יש קוד כפול?
2. האם הטיפול בשגיאות שלם?
3. האם הסגנון עקבי עם שאר הפרויקט?

פלט: רשימת ממצאים, מסודרת לפי חומרה.

מעכשיו, בכל שיחה, /review יריץ את כל הבדיקות האלו בלי לחזור עליהן ידנית.

הטיפ החשוב ביותר CLAUDE.md ו-slash commands הם לא "פיצ'רים מתקדמים" — הם הבסיס של עבודה יעילה עם Claude Code. ככל שתשקיעו יותר בהגדרתם, כך Claude יעבוד בדיוק לפי הציפיות שלכם — גם בשיחה הבאה, גם בפרויקט הבא.
