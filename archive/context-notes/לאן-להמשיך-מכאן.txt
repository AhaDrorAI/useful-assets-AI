**השאלות שצריך ממך**
1. מי הקהל הכי חשוב: אנשים בלי רקע טכני בכלל, יזמים, בעלי עסקים, אנשי מוצר/אופרציה, יוצרים, או קהל אחר?

8. אילו “מצבי משתמש” חייבים להיות באתר: פותח פרויקט חדש, תקוע בשגיאה, רוצה לשפר ריפו קיים, רוצה לבנות MVP, רוצה ללמוד workflows?

9. האם לכלול תבניות פרומפטים בעברית שאפשר להעתיק, כמו “תבדוק את הריפו”, “תבנה לי MVP”, “אל תכתוב קוד לפני תכנון”?

10. מה הטון הוויזואלי הרצוי: נקי ומקצועי, יותר playful, תחושה של operating manual, או “מפה אינטראקטיבית” של עולם הסוכנים?

11. האם הרשימה שנתת היא הבסיס המחייב, או שמותר לי להוסיף/להסיר/לשנות סדר אחרי בדיקה עדכנית של הריפואים?

12. מה ייחשב הצלחה בסוף: מדריך שאפשר לפרסם, אתר רץ, skill עובד, או מסמך אסטרטגי שממנו נבנה אחר כך?

**מה הייתי הופך ל־Skill**
שם אפשרי: `ai-project-launchpad`

ה־Skill לא יהיה “מדריך על Claude Code”. הוא יהיה **מלווה התנעה לפרויקט**.

הוא יופעל כשאתה כותב דברים כמו:
- “I want to start a new project”
- “Create a repo from this idea”
- “Turn this PRD into a working app”
- “Set up my Claude Code workflow”
- “I’m stuck and need to regain momentum”

והוא יכריח את הסוכן לעבוד כך:
`clarify → PRD → SPEC → repo setup → scaffold → first working slice → verify → handoff`

**הדבר הכי חשוב**
לא הייתי מנסה לגרום לך “לעבוד לפי כל ההמלצות”. זה מתכון לאשמה.  
הייתי בונה לך מערכת שבה יש רק שלושה כפתורים מנטליים:

- **Start**: פתח פרויקט נכון
- **Continue**: המשך מהנקודה האחרונה
- **Rescue**: אני תקוע, תחזיר אותי למסילה

זה יכול להיות באתר שלך כעמוד מרכזי, ובמקביל כ־Skill שהסוכן באמת מפעיל.

השאלה היחידה עכשיו: האם ה־Skill הראשון שנבנה צריך להיות **“מחשב חדש + התקנות”** או **“פתיחת פרויקט חדש מ־PRD/SPEC עד ריפו עובד”**?

ברור. אז ה־MVP הראשון הוא:

**New Machine For Claude Code**
חבילת התנעה למחשב חדש שכוללת:
1. עמוד אתר נעים וברור למשתמש.
2. מדריך Markdown מפורט.
3. Skill בשם למשל `new-machine-bootstrap`.
4. בהמשך: אותו מבנה יתרחב ל־`new-project-from-prd`.

אני אכניס את **Bun** לשכבת הפרויקט כבחירה מומלצת ומהירה, לצד `npm/pnpm`, עם כלל פשוט: בפרויקט חדש אפשר לבחור Bun; בפרויקט קיים לא מחליפים package manager בלי לבדוק lockfile ו־CI.

**מבנה האתר**
העמוד יהיה בנוי כ־checklist רגוע:

- “מה אני מתקין קודם?”
- “מה חובה ומה אופציונלי?”
- “איך יודעים שזה עובד?”
- “מה עושים אם משהו נשבר?”
- “איזה סטאק לבחור לפרויקט חדש?”
- “כפתור: תן לי פקודת Bootstrap למחשב שלי”

**שכבות ההתקנה**
- בסיס: Git, GitHub CLI, VS Code/Cursor, shell נוח.
- Runtime: Node LTS, Bun, Python/uv.
- Agents: Claude Code, Codex CLI.
- Project tools: Supabase CLI, Netlify CLI, Playwright.
- Workflow: Superpowers, AGENTS.md/CLAUDE.md templates.
