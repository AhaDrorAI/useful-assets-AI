לפני המדריכים המתקדמים

מכינים מחשב חדש לעבודה עם Claude Code ו-Codex

מדריך יד ביד לאדם שלא פתח טרמינל מעולם: מה להוריד, איפה ללחוץ, איזו פקודה להדביק, ומה אמור להופיע כשהכול עובד.

יש כבר באתר מדריך Claude Code ומאגר ריפואים. העמוד הזה מגיע לפניהם. הוא מלמד את הבסיס: טרמינל, התקנות, בדיקות הצלחה, וחיבור GitHub. אחרי זה אפשר להמשיך ל-מדריך Claude Code המלא ול-מאגר הריפואים.
1פותחים טרמינל
לומדים איפה מדביקים פקודות ומה הן עושות.
2מורידים בסיס
עורך קוד, Git, GitHub CLI ו-Node.
3מחברים סוכנים
מתקינים Claude Code ו-Codex ובודקים שהם עובדים.
4ממשיכים לפרויקט
רק אחרי שהבסיס עובד עוברים ל-PRD, SPEC וריפו חדש.

מה זה טרמינל?

הסבר בלי ז׳רגון

טרמינל הוא חלון טקסט שמדבר עם המחשב

במקום ללחוץ על כפתורים, מדביקים שורת טקסט ולוחצים Enter. השורה הזו נקראת פקודה.

לדוגמה, הפקודה הזו שואלת את המחשב איזו גרסה של Node מותקנת:

node --version

אם הכול תקין, תקבל תשובה קצרה כמו v24.15.0. המספר יכול להיות שונה וזה בסדר.

Windows

איך פותחים טרמינל ב-Windows

  1. לחץ על Start.
  2. כתוב PowerShell.
  3. פתח את Windows PowerShell או Terminal.
  4. אל תבחר Run as Administrator אלא אם מדריך רשמי מבקש זאת.
Mac

איך פותחים טרמינל ב-Mac

  1. לחץ Command + Space.
  2. כתוב Terminal.
  3. לחץ Enter.
כלל בטיחות ראשון אם פקודה כוללת מילים כמו delete, remove, rm, reset, force או מבקשת מפתח סודי, לא מריצים לפני שמבינים למה.

שלבי ההתקנה

שלב 1

התקן עורך קוד

עורך קוד הוא המקום שבו תראה את קבצי הפרויקט. גם אם הסוכן כותב את הקוד, אתה צריך מקום לפתוח את התיקייה.

  • הורדת VS Code - הבחירה הפשוטה להתחלה.
  • הורדת Cursor - עורך קוד עם יכולות AI מובנות.
שלב 2

התקן Git

Git שומר היסטוריה של הפרויקט ומאפשר לחזור אחורה אם משהו השתבש.

הורדת Git מהאתר הרשמי

אחרי ההתקנה פתח טרמינל חדש והריץ:

git --version

הצלחה: מופיעה שורה שמתחילה ב-git version.

שלב 3

התקן GitHub CLI והתחבר

הכלי הזה מאפשר לסוכן לעבוד מול GitHub: ליצור ריפו, לבדוק חיבור, ובהמשך לפתוח PR.

הורדת GitHub CLI

gh --version

אחר כך התחבר:

gh auth login

בדרך כלל בוחרים GitHub.com, אחר כך HTTPS, ואז Login with a web browser.

gh auth status

הצלחה: כתוב שאתה מחובר ל-GitHub.

שלב 4

התקן Node.js

Node מריץ הרבה פרויקטים מודרניים, ו-npm שמגיע איתו מאפשר להתקין כלים כמו Codex.

הורדת Node.js - בחר גרסת LTS.

node --version
npm --version
שלב 5

התקן Claude Code

דוקומנטציה רשמית של Claude Code

Windows PowerShell:

irm https://claude.ai/install.ps1 | iex

Mac:

curl -fsSL https://claude.ai/install.sh | bash

בדיקה:

claude --version
claude doctor

התחברות ראשונה:

claude
Claude Login

כניסה ל-Claude Code ואפשרויות שימוש

בפעם הראשונה שמריצים claude, משלימים התחברות בדפדפן לחשבון Anthropic. בתוך Claude Code אפשר גם להריץ /login אם צריך להתחבר מחדש.

אפשר לעבוד בשלוש דרכים עיקריות:

  • Terminal CLI: מריצים claude מתוך תיקיית הפרויקט.
  • Desktop app: מורידים מ-claude.com/download, נכנסים לחשבון, ואז פותחים את לשונית Code.
  • Web: משתמשים ב-claude.ai/code למשימות Code בענן או להפעלה דרך Remote Control.

מצבי פתיחה שימושיים:

claude
claude --continue
claude --permission-mode plan
claude --permission-mode acceptEdits
זהירות עם מצב מסוכן claude --dangerously-skip-permissions זהה ל---permission-mode bypassPermissions. הוא מדלג על בקשות אישור והרבה בדיקות בטיחות. משתמשים בו רק בסביבה מבודדת כמו container, VM או dev container, לא על המחשב הראשי של תלמיד מתחיל.
שלב 6

התקן Codex CLI

מקור רשמי: openai/codex

npm install -g @openai/codex

בדיקה:

codex
Codex Login

כניסה ל-Codex ואפשרויות שימוש

בפעם הראשונה שמריצים codex, Codex יבקש התחברות. אפשר להתחבר עם חשבון ChatGPT מתאים או עם API key.

codex login
codex login status
codex logout

אם הדפדפן לא נפתח, למשל בשרת מרוחק:

codex login --device-auth

אם משתמשים ב-API key, לא מדביקים מפתח ישירות לצ'אט. משתמשים במשתנה סביבה:

printenv OPENAI_API_KEY | codex login --with-api-key

אפשר לעבוד עם Codex בכמה דרכים:

  • Terminal CLI: מריצים codex מתוך תיקיית הפרויקט.
  • Desktop app: מריצים codex app. ב-macOS זה פותח את Codex Desktop או מתחיל התקנה; ב-Windows הוא יציג את הנתיב לפתיחה אחרי התקנה.
  • Web: משתמשים ב-chatgpt.com/codex אם התוכנית שלך כוללת גישה ל-Codex.

מצבי פתיחה שימושיים:

codex
codex --model gpt-5.4
codex --sandbox workspace-write --ask-for-approval on-request
codex --search
זהירות עם מצב מסוכן codex --dangerously-bypass-approvals-and-sandbox או codex --yolo מריצים פקודות בלי אישורים ובלי sandbox. משתמשים בזה רק בסביבת sandbox חיצונית ומבודדת.
שלב 7

התקן Bun

Bun הוא כלי מהיר להרצת פרויקטים חדשים. לא צריך להבין אותו לעומק עכשיו; מספיק לדעת שהוא שימושי בהמשך.

דוקומנטציה רשמית של Bun

Windows PowerShell:

powershell -c "irm bun.sh/install.ps1|iex"

Mac:

curl -fsSL https://bun.com/install | bash
bun --version
שלב 8

כלים לפרויקטים ספציפיים

לא חייבים להתקין הכול עכשיו. אלו כלים לפי צורך:

  • Supabase CLI לפרויקטים עם Database/Auth.
  • Netlify CLI לפרסום אתרים.
  • Playwright לבדיקות דפדפן.

מה לתת לסוכן אחרי ההתקנה

I am setting up this computer for AI coding work.

Please help me verify the setup slowly and safely.

I am not a developer, so explain every step in plain language.

Do not install anything before showing me:
1. What you are checking
2. Why it matters
3. The exact command
4. What a successful result should look like
5. What could go wrong

Check these tools:
- Git
- GitHub CLI
- Node and npm
- Bun
- Claude Code
- Codex CLI

After each check, wait for the result before moving to the next step.
